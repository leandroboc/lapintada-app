# Resumen del Proyecto y Estado Actual para Asistente de IA

## 1. Objetivo del Proyecto

Crear un chatbot de IA local para una empresa. El chatbot debe responder preguntas basándose únicamente en la información proporcionada en documentos de texto (`.txt`).

**Tecnologías clave:**
- **Backend:** Python con FastAPI.
- **IA Local:** Ollama con el modelo `gemma:2b`.
- **Lógica RAG (Retrieval-Augmented Generation):** LangChain.
- **Base de Datos Vectorial:** ChromaDB (en memoria por ahora).
- **Embeddings:** `all-MiniLM-L6-v2`.

## 4. Estado del Código (`main.py`)

Para poder verificar que la lógica de la IA es correcta, se ha modificado `main.py`. **Ya no es un servidor FastAPI.** Ahora es un script que, al ejecutarse, realiza los siguientes pasos:
1. Carga los documentos de la carpeta `data`.
2. Los procesa y crea una base de datos vectorial en memoria.
3. Configura la cadena RAG con el modelo `gemma:2b` de Ollama.
4. Ejecuta dos preguntas de prueba:
    - Una cuya respuesta está en `info_empresa.txt` ("¿Cuál es el horario de atención?").
    - Una cuya respuesta NO está en el contexto ("¿De qué color es el cielo?").
5. Imprime los resultados en la consola.

Esto permite probar toda la cadena de IA sin depender de un entorno virtual funcional o un servidor.
